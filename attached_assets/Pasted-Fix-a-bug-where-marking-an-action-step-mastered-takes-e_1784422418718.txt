Fix a bug where marking an action step "mastered" takes effect immediately during draft autosave instead of only at final observation submission. In artifacts/api-server/src/routes/observations.ts, both the POST handler (around lines 573-582) and the PUT handler (around lines 814-823) apply the `masterActionStepId` → action step status="mastered" transition unconditionally, regardless of whether the request's own `status` field is "draft" or "published". Only the walkthrough/rescore-queue logic in this file is currently gated on an `isPublishing` check — the action-step mastery transition needs the exact same gating: only perform the actual database update (status="mastered", masteredAt=now, masteredByEmployeeId, masteredDuringObservationId) when the observation write is publishing (status transitioning to or already "published"), not on draft saves. On a draft save, accept `masterActionStepId` in the payload as before (so the intent isn't lost if the draft is resumed later) but treat it as a no-op — do not touch the action_steps table and do not reject with a 400 for "already mastered" on drafts.

Both the mobile app (artifacts/catalyst-mobile/src/pages/observation.tsx) and the dashboard app (artifacts/catalyst-dashboard/src/components/NewObservationModal.tsx) send `masterActionStepId` on every autosave tick while editing a draft — after this fix, those autosave calls become harmless no-ops until the user actually submits, so no client-side changes should be needed, but verify this by re-reading both autosave call sites to confirm they don't have any client-side logic that also assumes immediate mastery (e.g. optimistically marking the local action-step state as mastered before the observation is actually published — if so, that optimistic update also needs to move to fire only after a successful publish).

Add an integration test that: (1) creates an observation as a draft with masterActionStepId set — assert the target action step's status is still "open" afterward; (2) publishes that same draft (or a subsequent PUT with status="published") — assert the action step is now "mastered" with the correct masteredAt/masteredByEmployeeId/masteredDuringObservationId. Also add a test that multiple draft-status autosaves carrying the same masterActionStepId do not error and do not double-apply anything.
